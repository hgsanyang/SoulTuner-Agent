# SoulTuner 用户功能与使用指南 (Feature Walkthrough)

欢迎使用 **SoulTuner-Agent**！它不是传统的"搜索-播放"音乐软件，而是一款具备**全双工对话**、**长时记忆感知**和**高拟真情绪理解**的 AI 音乐伴游智能体。

本指南将带您完整了解系统的每一项功能、交互细节与使用技巧。

---

## 1. 首页与场景引导

启动应用后，您将看到精心设计的 **欢迎界面 (WelcomeScreen)**，包含多张场景推荐卡片：

- 🎧 **深夜听歌** — "安静的夜晚，来点治愈系的音乐"
- 🏃 **运动健身** — "跑步健身，需要节奏感强的音乐"
- 🚗 **驾车出行** — "开车路上需要公路氛围感"
- ☕ **工作学习** — "适合专注工作时的背景音乐"
- 🌧️ **雨天治愈** — "下雨天想听温暖柔和的曲子"

**点击任意卡片**即可直接进入对话推荐页面，卡片上的文字会自动填入对话框作为您的第一句查询。当然，您也可以忽略卡片，直接在对话框中输入任何想法。

## 2. 核心对话推荐

### 2.1 五种对话姿势

SoulTuner 支持 5 种不同维度的自然语言对话：

| 类型 | 示例 | 系统行为 |
|------|------|---------|
| **纯情绪/氛围** | "我今天心情特别差，想一个人静一静" | 触发 Vector RAG。将情绪转化为英文声学描述（HyDE），跨模态检索 BPM 缓慢、音调低沉的治愈系音乐 |
| **场景+主观体验** | "跑步健身中，需要那种重低音砸耳朵、让我热血沸腾停不下来的！" | 触发 Hybrid RAG。场景标签精确筛选 + 声学振感向量匹配 |
| **干巴巴标签组合** | "来点周杰伦的浪漫情歌" / "英文摇滚" | 触发 GraphRAG。图谱精确匹配歌手、流派、情绪标签 |
| **时效性资讯** | "最近抖音最火的新歌有什么？" | 触发 Web Search。SearxNG 联网搜索 + LLM 摘要 |
| **指定歌曲** | "帮我找《稻香》" | 触发 GraphRAG 精确查找。本地库没有会引导使用数据飞轮入库 |

### 2.2 推荐过程的实时可视

当您发送消息后，界面上会呈现完整的 Agent 思考过程：

1. **💭 思考动画 (ThinkingIndicator)** — 系统正在调用记忆、分析意图
2. **🎵 歌曲卡片逐首出现** — 每检索到一首就立刻实时推送至界面（SSE 流式）
3. **📝 推荐理由文本流式生成** — 像打字一样逐字出现，由 LLM 生成的拟人化电台独白

每次推荐最多返回 **15 首歌曲**，每首都附带选中理由。

### 2.3 返回场景卡片

如果您觉得当前推荐不是想要的方向，可以点击页面顶部的 **← 返回** 按钮，回到场景选择卡片，重新选择一个新的场景开始对话，无需刷新页面或重启会话。

## 3. 歌曲卡片交互 (SongCard)

每首推荐的歌曲都呈现为一个功能丰富的 **SongCard** 卡片，从左到右包含：

### 3.1 播放功能

- **点击封面图** → 播放/暂停当前歌曲
- 封面 hover 时显示播放/暂停遮罩图标
- 无封面时显示渐变占位图 + 音符图标
- 正在播放的歌曲标题变为绿色高亮

### 3.2 操作按钮（每张卡片右侧）

| 按钮 | 图标 | 功能 | 后端行为 |
|------|------|------|---------|
| **播放列表** | ☰+ | 将歌曲加入/移出全局播放队列 | 纯前端操作 |
| **加入本地** | ↓ | 触发数据飞轮，下载音频/封面/歌词并入库 Neo4j | `POST /api/acquire-song` |
| **❤️ 红心** | ♥ | 点赞/取消点赞 | `POST /api/user-event` (like/unlike) → Neo4j `LIKES` 关系 |
| **👎 不喜欢** | 👎 | 标记为不喜欢，从当前结果移除 | `POST /api/user-event` (dislike) → Neo4j `DISLIKES` 关系 |
| **🔖 收藏** | 书签 | 收藏到自定义歌单 | `POST /api/user-event` (save) → Neo4j `SAVES` 关系 |
| **✕ 移除** | × | 从当前推荐列表中移除（不影响后续推荐） | 纯前端操作 |

### 3.3 标签展示

卡片下方显示歌曲的标签信息：
- **流派标签**（灰色底色）：如 `Rock`、`Folk/Indie`
- **情绪标签**（绿色底色）：如 `平静`、`热血`
- **播放列表标记**（绿色文字）：如 `▶ 播放列表`

### 3.4 推荐理由

卡片底部以虚线分割显示每首歌的推荐原因，例如：
- "引擎检索来源: 知识图谱(GraphRAG) + 语义向量(Neo4j Vector) 🔥双引擎交叉命中"

## 4. 点赞、收藏与不喜欢功能详解

### 4.1 ❤️ 点赞 (Like)

- **点击红心** → 立即在界面上变红，弹出 Toast 提示 `♥ 已添加到「我的喜欢」`
- **后端行为**：发送 `like` 事件 → Neo4j 创建 `(User)-[:LIKES {weight: 1.0}]->(Song)` 关系
- **再次点击** → 取消点赞（unlike），红心变空
- **后端行为**：发送 `unlike` 事件 → Neo4j 删除 `LIKES` 关系
- **对推荐的影响**：点赞的歌曲成为 Graph Affinity 的锚点，与它图谱距离近的歌曲会在后续推荐中获得更高评分

### 4.2 🔖 收藏到歌单 (Collection)

- **点击书签图标** → 弹出歌单选择下拉面板
- 选择一个歌单 → 歌曲加入该歌单，Toast 提示 `✓ 已添加到「深夜敲代码专用」`
- **后端行为**：发送 `save` 事件 → Neo4j 创建 `(User)-[:SAVES {weight: 0.8}]->(Song)` 关系
- 系统预设了两个歌单：`深夜敲代码专用`、`雨天治愈系`，用户也可自行创建新歌单

### 4.3 👎 不喜欢 (Dislike)

- **点击不喜欢** → 歌曲从当前推荐结果中立即移除，Toast 提示 `👎 已标记为「不喜欢」，后续推荐将过滤此歌曲`
- **后端行为**：
  1. 发送 `dislike` 事件 → Neo4j 创建 `(User)-[:DISLIKES]->(Song)` 关系
  2. **自动清理矛盾**：如果之前有 `LIKES` 或 `SAVES` 关系，会被自动删除
- **对推荐的影响**：被 dislike 的歌曲在后续所有推荐中被**永久过滤**（DISLIKES 黑名单机制，Step 2.5）
- **撤销**：可在个人音乐库的"不喜欢"列表中撤销 → `DELETE /api/disliked-songs`

### 4.4 全局 Toast 通知

所有操作都会在页面顶部中央弹出一个毛玻璃风格的 Toast 通知，2.5 秒后自动消失。

## 5. 全局音乐播放器 (GlobalPlayer)

页面底部固定的吸底播放器支持以下功能：

### 5.1 基础控制

| 功能 | 说明 |
|------|------|
| **播放/暂停** | 点击中央按钮控制播放 |
| **上一首/下一首** | 根据播放模式切换 |
| **进度条** | 拖动定位到指定播放位置 |
| **音量控制** | 滑块调节音量（0-100%） |

### 5.2 播放模式

点击播放模式按钮循环切换：

| 模式 | 行为 |
|------|------|
| **顺序播放** (`sequence`) | 按队列顺序播放，末尾回到队首 |
| **随机播放** (`random`) | 随机选择下一首 |
| **单曲循环** (`loop`) | 当前歌曲结束后重新播放 |

### 5.3 全屏展开

点击播放器上方的展开按钮，进入全屏模式：
- 大尺寸专辑封面
- 完整歌词展示（如有 `.lrc` 歌词文件）
- 粒子星空动画背景 (StarryBackground)

### 5.4 播放队列管理

- 在 SongCard 上点击"播放列表"按钮添加/移除歌曲
- 全局 `addAllToQueue` 可一次性将整批推荐加入队列
- 队列支持去重（基于 `title_artist` 唯一键）

### 5.5 隐式行为采集

播放器在后台自动采集用户行为，无需手动操作：

| 行为 | 触发条件 | 后端事件 |
|------|---------|---------|
| **完整播放** | 歌曲播放完毕 | `full_play` → Neo4j `LISTENED_TO` 关系（play_count++） |
| **快速跳过** | 播放未满 30 秒即切歌 | `skip` → Neo4j `SKIPPED` 关系（skip_count++） |
| **单曲循环** | 单曲循环模式播放 | `repeat` → Neo4j `LIKES` 关系增强权重 |

## 6. 沉浸式音乐旅程 (Music Journey)

### 6.1 入口

在侧边栏点击 **🗺️ 音乐旅程** 进入。

### 6.2 使用方式

1. **输入故事**：描述一段场景或心情故事，例如：
   > "我今晚经历了失恋，现在一个人开车在雨中回老家，大概需要 60 分钟"

2. **可选设置**：
   - 地点、天气/活动、总时长
   - 自定义情绪曲线节点（时间 + 情绪 + 强度）

3. **LLM 故事拆解**：系统将故事拆解为 3-6 个有起承转合的**情绪片段**：
   ```
   片段1: 压抑沉闷 (25%) → 慢节奏、暗色调钢琴
   片段2: 情感宣泄 (25%) → 中速摇滚/后摇
   片段3: 逐渐平静 (30%) → 民谣、轻电子
   片段4: 治愈释怀 (20%) → 温暖 acoustic、indie folk
   ```

4. **流式检索与播放**：每个片段独立检索 2-4 首歌曲，SSE 实时推送：
   ```
   segment_start → song → song → song → segment_complete
     → transition_point
   segment_start → song → song → segment_complete → ...
   ```

5. **无缝连续播放**：片段间自然过渡，像电影配乐一样陪伴整段旅程。

### 6.3 分享卡片

每个片段生成可分享的**音乐卡片**（`generateMusicCard`），包含标题、歌手、情绪标签和话题标签。

## 7. 个人音乐库 (Library)

### 7.1 我的喜欢

- 展示所有点赞过的歌曲，按时间倒序排列
- 支持播放、取消喜欢操作
- 点击"播放我收藏的音乐"（对话框输入）触发 `recommend_by_favorites` 意图 → 返回 Seeds + Discoveries 混合推荐

### 7.2 我的歌单

- **预设歌单**：`深夜敲代码专用`、`雨天治愈系`
- **创建歌单**：自定义歌单名称
- **管理歌曲**：长按可从歌单中移除
- 歌单数据持久化在 `localStorage`，收藏事件同步到后端

### 7.3 不喜欢管理

- 展示所有标记为"不喜欢"的歌曲
- **撤销不喜欢**：点击"移除"按钮 → `DELETE /api/disliked-songs` → 从 Neo4j 删除 `DISLIKES` 关系
- 撤销后，该歌曲可重新出现在推荐结果中

## 8. 用户画像系统

### 8.1 显式画像设置

在 **用户画像面板 (UserProfilePanel)** 中，您可以直接设置：

| 维度 | 可选项 |
|------|-------|
| **流派偏好** | Pop / Rock / Jazz / Electronic / Hip-Hop / R&B / Classical / Folk / Metal / Blues |
| **情绪倾向** | 开心 / 宁静 / 浪漫 / 怀旧 / 热血 / 治愈 |
| **场景偏好** | 运动 / 通勤 / 学习 / 睡前 / 派对 |
| **语言偏好** | 中文 / 英文 / 日语 / 韩语 / 粤语 |
| **排除流派** | 讨厌的流派（推荐时硬过滤） |
| **排除歌手** | 讨厌的歌手 |

保存后**双写**到：
- Neo4j User 节点属性（持久化）
- GraphZep 长期记忆（可被 Agent 召回引用）

### 8.2 隐式画像积累

您在听歌过程中的每一次操作都在无声优化画像：

| 用户动作 | 画像影响 |
|---------|---------|
| ❤️ 点赞一首「民谣」 | `favorite_genres += "folk"` |
| 🔁 反复循环一首歌 | 该歌成为最强偏好锚点 |
| ⏭️ 连续跳过「电子舞曲」 | skip_count ≥ 3 时降权 |
| 👎 不喜欢「重金属」 | 后续推荐永久过滤该歌 + 类似风格降权 |
| 💬 对话中说"我不爱听电子乐" | LLM 提取 → `avoid_genres += "electronic"` |

### 8.3 数据同步机制

```
前端启动 → syncFromBackend()
  ├── GET /api/liked-songs    → 合并到本地 likedSongs
  └── GET /api/disliked-songs → 覆盖本地 dislikedSongs
```

确保前端与后端数据一致，避免多端操作导致的状态不同步。

## 9. 运行时设置面板 (Settings)

设置面板允许您在不重启系统的情况下调整核心参数：

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| **LLM 供应商** | DeepSeek / Gemini / 通义千问 / 本地模型 | DeepSeek |
| **联网搜索** | 开启/关闭 Web Search | 开启 |
| **RRF 向量权重** | 向量检索在融合中的权重 | 0.7 |
| **RRF 图谱权重** | 图谱检索在融合中的权重 | 0.3 |
| **MMR λ** | 多样性参数（越高越相关，越低越多样） | 0.7 |
| **Graph Affinity 权重** | 个性化加权强度 | 0.3 |
| **每歌手最多推荐数** | Artist 多样性限制 | 3 |

## 10. 数据飞轮操作

### 10.1 "加入本地"按钮

每张 SongCard 上的下载按钮支持一键入库：

1. 点击 **↓ 加入本地** → 按钮变为旋转加载动画
2. 后端从网易云 API 搜索并下载：
   - 📀 音频文件 (.mp3)
   - 🖼️ 专辑封面 (.jpg)
   - 📝 歌词文件 (.lrc)
3. 自动触发 DataFlywheel V2 管线：
   - M2D-CLAP + OMAR-RQ 双模型提取嵌入向量
   - LLM 零样本推断标签（流派/情绪/场景/BPM）
   - 写入 Neo4j 图谱
4. 完成后按钮变为 **✓ 已加入本地**（绿色对勾）
5. 该歌曲立刻可被后续检索找到并播放

### 10.2 对话式获取

在对话中说 "好的，帮我下载这些歌" → 系统识别 `acquire_music` 意图 → 批量下载入库。

1. 📝 先在用户画像面板中设置您的基本偏好
2. 🗣️ 对话框输入：*"我今天终于完成了大项目，需要一首能让我原地起飞的歌"*
3. ❤️ 对满意的歌曲点赞、收藏到歌单
4. 👎 对不合口味的歌曲标记不喜欢
5. 🔄 多轮对话后，系统会越来越懂你

### 11.2 进阶玩法

- 🗺️ 尝试音乐旅程：写一段 60 分钟的故事体验沉浸式听歌
- 🌐 时效性搜索：问 "最近有什么好听的新歌推荐给我"
- 📦 数据飞轮：发现好歌时点 "加入本地"，丰富你的个人曲库
